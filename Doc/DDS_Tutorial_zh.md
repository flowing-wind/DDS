## 1. DDS 到底是什么(原理)

DDS = **Direct Digital Synthesis,直接数字频率合成**。一句话:用纯数字的方式,
从一个固定的高速时钟里"算"出任意频率的波形,再送给 DAC 变成模拟信号。

传统的模拟振荡器靠 LC / 晶体谐振决定频率,想改频率要改电容电感。DDS 完全不同——
频率只是一个**数字**,改这个数字就改频率,而且改得又快又准、相位还连续。

### 1.1 三个核心部件

```
   ┌─────────────┐  每个时钟加一次   ┌──────────────┐   ┌─────┐
   │  调谐字      │ ───FWORD──────►  │ 相位累加器   │──►│ 查找 │──► 波形样点 ──► DAC
   │  FWORD(32位)│                   │  (32位)      │   │ 表  │
   └─────────────┘                   └──────────────┘   └─────┘
                                      每个 dds_clk 周期: phase += FWORD
```

1. **相位累加器(phase accumulator)**:一个 32 位寄存器,每来一个时钟(`dds_clk`,
   本工程 100 MHz)就加上一个固定值 `FWORD`。它在做的事,就是"匀速转圈"——把
   0 到 2³² 想象成一个圆(0°~360°),累加器的值就是当前相位。`FWORD` 越大,每步
   转得越多,转一圈越快,频率越高。加满溢出回绕,正好对应相位转过 360° 回到起点。

2. **波形查找表(LUT)**:相位(转到圆上的哪个角度)本身不是波形。把相位的高位拿去
   查一张表,表里存着"这个角度对应的正弦值是多少",就得到正弦样点。换张表 / 换个
   算法,就能得到方波、三角波等。本工程正弦用 4096 点的 1/4 波查找表。

3. **DAC**:把数字样点变成模拟电压。本工程用 14 位 DAC(AD9764)。

### 1.2 频率公式(最重要的一条)

```
    f_out = FWORD × f_dds / 2³²
```

- `f_dds` = DDS 时钟 = 100 MHz(本工程)
- `2³²` = 相位累加器满量程(32 位)
- `FWORD` = 你写进去的调谐字

**直觉**:相位累加器要累加 `2³²` 才转完一整圈(输出一个完整周期)。每个时钟加 `FWORD`,
所以转一圈要 `2³²/FWORD` 个时钟,也就是 `2³²/FWORD × (1/f_dds)` 秒一个周期,取倒数就是上式。

反过来,要某个频率 `f_out`,调谐字:

```
    FWORD = round(f_out × 2³² / f_dds)
```

**好在你不用手算**:串口敲 `freq 1M` 固件会自动换算。手算只是为了理解。

### 1.3 为什么频率能非常精细

`FWORD` 每变化 1,频率就变 `f_dds/2³²` = 100 MHz / 4.29e9 ≈ **0.0233 Hz**。这就是
**频率分辨率**——你能以 23 毫赫的步进设定频率。这是 DDS 相对模拟振荡器的杀手锏。

## 2. 本 DDS 的整体结构

```
 上位机(PC 串口)
      │ ASCII 命令 "freq 1M" ...
      ▼
 STM32F103 ──SPI(16位, Mode0, 9MHz)──►  FPGA 里的 DDS 核 ──14位并口──► AD9764 DAC ──► SMA 输出
      │                                        ▲
      └──把命令翻译成寄存器写入───────────────────┘
```

- **STM32** 跑一个串口命令行,把你的命令翻译成对 FPGA 内部寄存器的 SPI 读写。你几乎
  不用关心 SPI 和寄存器细节,敲命令就行。需要底层控制时可用 `reg` 命令直接读写寄存器。
- **FPGA DDS 核**:单一时钟域(`dds_clk` = 100 MHz)。内部有相位累加器、扫频引擎、
  查找表、幅度乘法器、直流偏置、输出格式转换。所有可调项都是寄存器。
- **双通道**:两路完全独立的 DDS,共用一根片选,靠命令字里的一个 bit 区分。命令 `ch 0` /
  `ch 1` 切换当前操作哪一路。

### 关键概念:原子提交(atomic commit)

一个 SPI"帧"里写的所有参数,**在片选拉高的瞬间一起生效**。这意味着:把 32 位频率字
分成两个 16 位写下去,DAC 绝不会在中间出现一个错误的过渡频率。你不用担心"写一半被看到"。

## 3. 频率

**命令**:`freq <hz>`

数字支持多种写法,都合法:

```
dds> freq 1000000     # 纯数字
dds> freq 1e6         # 科学计数法
dds> freq 1M          # SI 后缀:M=1e6, k=1e3, m=1e-3, u=1e-6
dds> freq 1.5k        # 1500 Hz
dds> freq 12.5M       # 12.5 MHz
```

设完后固件会回读实际频率(受 23.3 mHz 分辨率量化后的真实值)给你确认。

**上限(奈奎斯特)**:理论上 `f_out < f_dds/2 = 50 MHz`。但对这块带 32 MHz 贝塞尔滤波器
的 AD9764 模块,实际**建议 ≤ 20 MHz**,再高镜像信号会漏出来使波形抖动(见 §14)。
固件会把超过奈奎斯特的请求钳到 50 MHz。

**分辨率**:0.0233 Hz。你可以设 `freq 1.000023M` 这种精细值。

## 4. 幅值与电压

这是初学最容易混淆的部分,分两级理解。

```
   最终输出电压  =  硬件满量程 Vpp_fs   ×   AMP(数字缩放 0~1.0)
                   ↑ 由 DAC 参考电压定      ↑ 由 amp 命令定
```

### 4.1 硬件满量程:参考电压

`AMP = 1.0` 时输出多少伏,由 AD9764 的**参考电压**决定(不是只能靠电位器)。这块模块
提供三种设置(详见 AN01 §5):

| 设置方式 | 参考电压 | 满量程 Vpp(高阻) |
|---|---|---|
| J3/J5 跳线装上(**出厂默认**) | 内部 1.2 V | ≈ 6 Vpp |
| 跳线移到电位器位 | VR1/VR2 手调 0.1~1.25 V | ≈ 0.5 ~ 6 Vpp |
| J2/J4 的"3"脚外部基准输入 | 外灌 0.1~1.25 V(可软件控制) | ≈ 0.5 ~ 6 Vpp |

**"参考电压最大值" = 1.25 V**(对应满量程约 6 Vpp)。默认内部基准 1.2 V。

> **50 Ω 减半**:输出串了 50 Ω 电阻。接示波器**高阻(1 MΩ)**看到全幅(~6 Vpp);接
> **50 Ω 终端/负载**只有一半(~3 Vpp)。测量和校准时务必固定用同一种负载。

### 4.2 数字缩放:AMP

**命令**:`amp <0..1>`

```
dds> amp 1.0     # 满幅
dds> amp 0.5     # 半幅
dds> amp 0       # 静音(仍可叠加直流偏置,见 §5)
```

AMP 是 Q1.15 定点数字乘法器,细腻、快、无毛刺,适合做幅度调制 / 软启停 / 微调。
**代价**:缩得越小分辨率越低(`amp 0.1` 只剩约 10.7 个有效比特),噪声地板相对抬高。

**分工建议**:粗调用硬件参考电压(定档位),细调 / 动态调用 AMP。

### 4.3 怎么得到精确的比如 2 Vpp

标称满量程 6 V 有几 % 容差,直接算 `2/6` 不够准。做**一次性校准**:

```
dds> wave sine
dds> amp 1.0
dds> freq 100k
dds> on
```

用示波器量出**实测**满量程 `Vpp_fs`(比如量到 5.90 V)。然后:

```
    amp 值 = 目标Vpp / Vpp_fs = 2.0 / 5.90 ≈ 0.339
```

```
dds> amp 0.339      # 输出精确 2 Vpp
```

这样把硬件容差也校掉了。换了负载(50Ω↔1MΩ)或改了参考电压,要重新校一次。

若长期只用 2 Vpp 且要满分辨率:改用外部基准把 `Vpp_fs` 直接压到 2 V,再 `amp 1.0`,
就能用满 14 位。

## 5. 相位与直流偏置

### 5.1 相位偏移

**命令**:`phase <deg>`

给波形叠加一个相位偏移,不改频率。分辨率 360°/65536 ≈ 0.0055°。

```
dds> phase 90      # 超前 90°
dds> phase 180
```

主要用途:双通道之间对齐相位(做 I/Q、正交信号),或让波形从特定相位起步。

### 5.2 直流偏置(DC Offset)

**命令**:`offset <-1..1>`(满量程的分数,-1 = 负满,+1 = 正满)

在幅度缩放**之后**叠加一个直流电平,整个模拟链是直流耦合的,所以偏置真的会出现在
SMA 输出端。

```
dds> offset 0.25     # 波形整体抬高 25% 满量程
dds> offset 0        # 无偏置(默认)
```

**纯直流电压源玩法**:`amp 0` + `offset x`,输出就是一个可编程的平直直流电平
(默认基准下约 −3 V ~ +3 V)。

> 注意别超范围:`|AMP·满幅| + |offset|` 要留在 ±满量程内,否则数字饱和(削顶)。

## 6. 六种波形

**命令**:`wave <sine|cos|square|tri|ramp|user>`

| 波形 | 命令 | 说明 |
|---|---|---|
| 正弦 | `wave sine` | 4096 点 1/4 波查找表,起点 0、上升 |
| 余弦 | `wave cos` | 正弦超前 90°,起点在正峰 |
| 方波 | `wave square` | 占空比可调(见下),起点在正峰 |
| 三角波 | `wave tri` | 起点 −满幅、上升到 +满幅再回落 |
| 锯齿波 | `wave ramp` | 相位高位直接当输出,周期末回绕 |
| 用户波形 | `wave user` | 播放你写进 RAM 的任意波形(见 §10) |

### 方波占空比

**命令**:`duty <percent>`,只对方波有效。

```
dds> wave square
dds> duty 25       # 25% 占空比
dds> duty 50       # 对称方波(默认)
```

### 输出格式(一般不用动)

- `fmt <2c|ob>` — DAC 编码。**这块 AD9764 模块必须用 `ob`(offset binary,偏移二进制)**,
  也是复位默认。用错会让正弦被"劈成两半"(MSB 反转)——这正是之前那个波形 bug 的根因。
- `width <14|12>` — DAC 位宽,默认 14。
- `inv <on|off>` — 输出取反(整体上下翻转)。

## 7. 三种频率模式

DDS 不止能输出固定频率。频率的"来源"有三种模式:

| 模式 | 命令 | 频率来源 | 典型用途 |
|---|---|---|---|
| **定频 FIXED** | (默认,`freq` 即定频) | 固定 `FWORD` | 普通信号源 |
| **扫频 SWEEP** | `sweep ...` | 从 f1 线性扫到 f2 | 频响测试、Chirp、雷达 |
| **跳频 PROFILE** | `prof` / `profsel` | 4 组预设频率间跳变 | FSK/PSK 数字调制 |

下面两节分别讲扫频和跳频的原理与用法。

---

## 8. 扫频

### 8.1 原理

**扫频(Sweep / Chirp,啁啾)**:让输出频率随时间**线性**地从起始频率 `f1` 爬到终止
频率 `f2`。硬件里的做法是:在原来的相位累加器之外再加一个**频率累加器**——每隔
`RATE` 个时钟,就给当前调谐字加一个小步进 `FDELTA`,于是频率一格一格往上走:

```
   频率
   f2 ┤            ___________
      │          /
      │        /   ← 每 RATE 个时钟,频率 += FDELTA
   f1 ┤______/
      └──────────────────────► 时间
        步数 = (f2-f1)/FDELTA,总时长 = 步数 × RATE / f_dds
```

这整件事在 FPGA 硬件里自动完成,**不占 CPU**——你只要设好起止和时长,硬件自己扫。

### 8.2 用法

**命令**:`sweep <f1> <f2> <ms> [single|saw|updown]`

```
dds> wave sine
dds> sweep 100k 1M 10          # 100kHz→1MHz,10 毫秒扫完,扫一次
dds> start                     # 开始扫频
dds> stop                      # 中止,回到 f1
```

第 4 个参数是**扫频模式**(可选,默认 single):

| 模式 | 行为 |
|---|---|
| `single` | 扫一次,停在 f2 |
| `saw` | 锯齿:扫到 f2 立刻跳回 f1 重扫,循环 |
| `updown` | 三角:扫到 f2 再往回扫到 f1,来回乒乓 |

```
dds> sweep 1M 10M 5 saw        # 1→10MHz 反复扫,每次 5ms
dds> start
```

**约束**:`f2` 必须大于 `f1`(引擎只向上扫;要向下用 `updown`)。固件里的求解器会
自动算好 `FDELTA` 和 `RATE`,让实际时长落在你要的时间的 0.05% 以内。

**外部触发**(进阶):可配置成由 FPGA 的 `sweep_trig` 引脚上升沿触发,而不是 `start`
命令,用于和外部事件同步(见数据手册 §5.9)。

## 9. 跳频调制:FSK / PSK

### 9.1 原理

**FSK(Frequency-Shift Keying,频移键控)**:用两个(或多个)不同频率代表数字比特——
比如 1 MHz 代表 "0"、2 MHz 代表 "1"。发送方按数据流在这几个频率间**瞬间切换**,接收方
测频率就还原出比特。这是最基础的数字调制之一(蓝牙、老式 modem 都用过)。

**PSK(Phase-Shift Keying,相移键控)**:类似,但切换的是**相位**(比如 0° 代表 "0",
180° 代表 "1")。

本 DDS 用 **Profile(配置档)** 实现:预先在 4 个档里各存一组"频率 + 相位",然后
在这 4 个档之间瞬间切换。因为频率和相位都预存好了,切换发生在**一个采样周期内**,
且**完全不占 CPU**——这正是 FSK/PSK 需要的。

```
   Profile 0: f=1MHz, phase=0°     ┐
   Profile 1: f=2MHz, phase=0°     │  切换 profsel → 输出频率/相位瞬间跳变
   Profile 2: ...                  │
   Profile 3: ...                  ┘
```

### 9.2 用法

**第 1 步:编程各个 profile**。命令:`prof <0-3> <hz> [deg]`

```
dds> prof 0 1M          # 档0 = 1MHz, 相位 0°
dds> prof 1 2M          # 档1 = 2MHz
dds> prof 2 1M 180      # 档2 = 1MHz 但相位 180°(可做 PSK)
```

**第 2 步:切换档位**。命令:`profsel <0-3|pin>`

```
dds> wave sine
dds> on
dds> profsel 0          # 立刻切到档0 → 输出 1MHz
dds> profsel 1          # 立刻切到档1 → 输出 2MHz
```

你反复敲 `profsel 0/1`,就是在手动做 FSK。

**引脚驱动的零延迟 FSK**(进阶):

```
dds> profsel pin        # 档位改由 FPGA 的 hop_sel[1:0] 两根引脚决定
```

之后哪个档生效由硬件引脚电平实时决定,可以用外部数据流以采样率速度调制,STM32
完全不参与。这就是"零 CPU 负载的 FSK/PSK"。

## 10. 用户自定义波形

如果内置六种波形不够,可以把**任意 4096 点波形**写进 FPGA 的波形 RAM 播放。

**命令**:`user <sine|tri|pulse|stair>`(固件内置几种示例形状,写进 RAM 并切到 user)

```
dds> user pulse         # 窄脉冲(5% 占空)
dds> user stair         # 8 级阶梯波
dds> freq 10k
dds> on
```

内置示例:`sine`(半波整流正弦)、`tri`(对称三角)、`pulse`(窄脉冲)、`stair`(阶梯)。
它们是硬件波形发生器**做不出**的形状,专门用来演示 RAM 播放通路。

要放**你自己算的**波形,可在固件里改 `fill_user_wave()` 生成表,或用底层 API
`dds_load_user_wave_at()` / 直接 `reg` 写 `0x2000` 起的 RAM。一张表是一个完整周期,
任意频率下都每周期遍历一遍。

## 11. 双通道

两路独立 DDS,用 `ch` 命令切换当前操作对象:

```
dds> ch 0
dds> freq 1M
dds> wave sine
dds> on
dds> ch 1
dds> freq 2M
dds> wave square
dds> on
```

各通道有各自完整的一套寄存器,互不影响。

**关于两通道的相位关系**:通过 SPI 分两帧使能两个通道,时间上差几微秒,所以两个累加器
之间会有一个**固定的**相位差(不漂移,但不为零,且随频率变化)。要做严格锁相的 I/Q 对:
用示波器量出这个差,写进其中一路的 `phase` 补偿掉;或在 FPGA wrapper 里加一个同步信号
让两路同时离开复位(见 AN01 §9)。

| 命令 | 作用 |

| `freq <hz>` | 定频频率(`1M` `1e6` `1.5k` 都行) |

## 13. 从零点亮:一步步上手

按顺序做,每步都能验证:

```
1) 串口连上(115200 8N1),应看到开机横幅和 dds> 提示符

2) dds> id
   → 两个通道都要打印 ID 0x4453 VERSION 0x0200
   → 打不出?先停下查 SPI 接线,DAC 后面都免谈

3) dds> freq 1M
   dds> wave sine
   dds> fmt ob          # 确认偏移二进制(AD9764 必须)
   dds> on
   → 示波器 1MΩ 高阻应看到约 6 Vpp 干净正弦(50Ω 终端约 3 Vpp)

   dds> wave square
   dds> duty 25         # 25% 方波

   dds> wave sine

5) 扫频:
   dds> sweep 100k 2M 20 saw
   dds> start
   → 示波器上看到频率来回扫;dds> stop 停

   dds> prof 1 2M
   dds> wave sine
   dds> on
   dds> profsel 0       # 1MHz
   dds> profsel 1       # 2MHz —— 手动 FSK

7) 出问题:dds> status 看当前状态,dds> reset 清相位
```

## 14. 实践注意事项

理解这几条能帮你判断"波形对不对是我的问题还是极限":

- **奈奎斯特上限**:`f_out < f_dds/2 = 50 MHz`,这是绝对上限。低于它才谈得上重建。

- **镜像(alias)与滤波器**:DAC 输出天然含有 `k×f_dds ± f_out` 的镜像分量。本模块的
  32 MHz 贝塞尔滤波器为 100 MHz 时钟设计:输出 10 MHz 时第一个镜像在 90 MHz,被滤得干干
  净净。但输出超过 ~20 MHz 后镜像靠近通带,会看到波形抖动/失真——不是你程序错,是极限。
  **这也是为什么 `dds_clk` 一定要跑 100 MHz 而不是 50 MHz。**

- **sin(x)/x 滚降**:DAC 的零阶保持会让高频幅度自然下降(0.2×f_dds 处 −0.9 dB,
  0.5×f_dds 处 −3.9 dB)。要 20 MHz 以上平坦,得在模拟端或用 AMP 频率补偿表校正。

- **方波/三角波的边沿**:经贝塞尔滤波后边沿被限制在 ~11 ns 上升,1 MHz 方波很标准,
  3~5 MHz 以上方波角会明显变圆——这是重建滤波器的正常行为,不是 bug。

- **50 Ω 一致性**:测量/校准始终用同一种负载(高阻或 50Ω),否则幅值差一倍。

- **供电**:模块 5 V 供电(绝对最大,只有反接保护没有过压保护),用线性电源或滤波好的
  电源,开关电源纹波会抬高噪声地板。
